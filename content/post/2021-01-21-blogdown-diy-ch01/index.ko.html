웹북 blogdown: Creating Websites with R Markdown의 첫번째 장을 따라합니다.

패키지 설치

blogdown패키지를 설치하고 HUGO를 설치합니다.
그리고 설치된 HUGO의 버전을 확인합니다.

install.packages("blogdown", dependencies = TRUE)
blogdown::install_hugo()
blogdown::hugo_version()

R project 만들기

이곳을 참고하여 Github의 원격 저장소(Remote Repository)와 연결된 R Project를 만듭니다.

블로그 만들기

blogdown::new_site(theme = "kakawait/hugo-tranquilpeak-theme",
                   format = "toml")

blogdown:new_site()와의 차이점은 아래와 같습니다.

  • theme
    • default: theme = "yihui/hugo-lithium"
    • 변경 후: kakawait/hugo-tranquilpeak-theme
    • 해당 테마에 관한 정보는 이곳에서 확인할 수 있습니다.
  • format
    • default: format = "yaml"
    • 변경 후: format = "toml"
    • 우리가 선택한 테마의 구성(configuration)파일의 형식은 TOML입니다.
      예제사이트와 구성파일의 형식을 일치시키기 위해 기본값을 변경했습니다.

Git Bash를 이용하여 Github에 push

# Git Bash에서 실행
git add .
git commit -m "new-site"
git push origin main

R studio에서 제공하는 Git GUI를 이용하여 Git을 동작하면 R studio가 멈추는 현상이 나타납니다.
기본 테마의 경우 이런 문제가 발생하지 않으나 테마를 변경하여 사이트를 생성하면 이와 같은 문제가 나타나는 것으로 보입니다.
그 대신 Git 설치 시 따라오는 Git Bash를 이용하여 ’stage -> commit -> push’를 수행합니다.
R studio의 기본 터미널을 Git Bash로 변경하면 디렉토리를 변경할 필요가 없어 좀 더 편리하게 명령어를 입력하실 수 있습니다.
